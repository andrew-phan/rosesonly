Admin Logger

Version 1.0.1

Admin Logger is a great Magento extension which logs actions performed in the backend of your Magento store. With this extension, you can discover which administrator logged in, which pages they visited and what modifications made. You can view the detailed information on the history of actions with data and time. So if something goes wrong in your store, you can find out what could cause this problem.

Features

  • Ability to log 50+ groups of actions
  • Ability to log actions of all or specified admins
  • Ability to log all or specified groups of actions
  • Ability to log changes in details
  • Color highlight of changes while log details viewing
  • Automatic log clearing on cron run
  • Ability to export log in Excel/CSV file
  • Blog Pro (http://www.magpleasure.com/blog-pro.html) compatibility
  • Active Content Slider (http://www.magpleasure.com/active-content-slider.html) compatibility
  • Guest Book (http://www.magpleasure.com/guest-book.html) compatibilty
  • Intuitive interface
  • Easy 2-minutes installation
  • Meets Magento programming practices for most versions and customizations compatibility

Installation

Clear the store cache under var/cache and all cookies for your store domain. This step eliminates almost all potential problems. It's necessary since Magento uses cache heavily.

Backup Your Data

Backup your store database and web directory.

Download and Extract

Download and unzip extension contents on your computer and navigate inside the extracted folder.

Upload Files

If you use a different from default theme - be sure to rename app/design/frontend/default/default and skin/frontend/default/default folders to your store's values. Using your FTP client upload files app, js and skin directory to your store root. Depending on the module some directories mentioned may not exist in the package.

Configuration

After extenion installation you need to configure it. Open the System->Admin Logger->Configuration page.

  • Enabled
    Turn on/off logs
  • Users
    Define the actions of which users should be logged
  • Action Groups
    Define the groups of actions which will be logged
  • Save Details
    Define whether detailed information will be enabled in logs
  • Delete Log Records After X Days
    Define the time for the logs to be saved (in days). The 0 value means that time is not limited.

Reminder!

If you have already configured cron jobs for your Magento installation, then you can skip the following step. Read the following instructions on setting cron job for your Magento store: http://www.magentocommerce.com/wiki/1_-_installation_and_configuration/how_to_setup_a_cron_job Generally it would be enough to run in SSH console of your server: 

crontab -e

And insert the following line: 

*/3 * * * * php /<path-to-your-installation/cron.php

After that exit confirming save request.

Troubleshooting

After the extension installation the store gives an error, or blank page, or suggests to start Magento installation procedure.
Change the owner of the extracted extension files to the web server user and set 775 permissions on them. Clear the store cache and try again.
There is no MagPleasure extensions under my configuration section, or having the extension tab clicked I get a blank page, or Access Denied error.
Clear the store cache, browser cookies, logout and login again.
I've set up everything correctly, inserted the HTML code but there is nothing on that page.
Clear the store cache, clear your browser cache and domain cookies and refresh the page.
My configuration changes do not appear on the store.
Clear the store cache, clear your browser cache and domain cookies and refresh the page.
